# Homebrew Tap for `llmind-cli`

> **Status: deprecated.** The recommended install is now `pipx install
> 'llmind-cli[anthropic]'` (or `uv tool install â€¦`). Homebrew forces source
> builds for every Python wheel, which drags in a 3 GB toolchain (LLVM, Rust,
> Poppler, GnuTLS, NSS, GnuPG, â€¦) and routinely fails on packages like
> `pikepdf` whose PEP 517 build needs `pybind11` + `cmake`. The pipx/uv path
> uses prebuilt wheels and finishes in seconds.
>
> The formula and tap workflow below are kept for reference and for users who
> insist on `brew`. Note: the resource block in `llmind-cli.rb` is now stale
> relative to the slimmer `pyproject.toml` and would need
> `regenerate_resources.sh` to be re-run before the formula will install.

This directory holds the source-of-truth Homebrew formula for `llmind-cli`.
It is intended to be published from a separate **tap repo** so users can run:

```bash
brew tap dmitryrollins/llmind
brew install llmind-cli
```

The formula here lives next to the source so version bumps stay in sync with
the codebase. CI (or you, manually) copies it into the tap repo on each
release.

---

## One-time tap setup

1. **Create the tap repo on GitHub.** Name must be `homebrew-llmind`
   (Homebrew strips the `homebrew-` prefix, so the user-facing name is
   `dmitryrollins/llmind`):

   ```bash
   gh repo create dmitryrollins/homebrew-llmind --public --description "Homebrew tap for LLMind"
   ```

2. **Initialize the tap layout:**

   ```bash
   git clone git@github.com:dmitryrollins/homebrew-llmind.git
   cd homebrew-llmind
   mkdir Formula
   ```

3. **Copy the formula in:**

   ```bash
   cp /path/to/LLMind/packaging/homebrew/llmind-cli.rb Formula/llmind-cli.rb
   ```

   Don't push yet â€” the resource block is still empty (see next section).

## Cutting a release

Each new CLI release needs four things in lockstep:

1. **Bump the version** in [`llmind-cli/llmind/__init__.py`](../../llmind-cli/llmind/__init__.py)
   and [`llmind-cli/pyproject.toml`](../../llmind-cli/pyproject.toml).
2. **Tag and release** the monorepo. Use a CLI-specific tag prefix so a future
   `llmind-app` release doesn't collide:

   ```bash
   git tag cli-v0.1.0
   git push origin cli-v0.1.0
   gh release create cli-v0.1.0 --title "llmind-cli 0.1.0" --notes "..."
   ```

3. **Update `url`, `version`, and `sha256`** in `llmind-cli.rb`. The tarball
   URL is GitHub's auto-generated archive for the tag. Compute its SHA256:

   ```bash
   curl -sL https://github.com/dmitryrollins/LLMind/archive/refs/tags/cli-v0.1.0.tar.gz \
     | shasum -a 256
   ```

4. **Regenerate the Python resources block.** Inside the tap repo:

   ```bash
   brew tap dmitryrollins/llmind /path/to/homebrew-llmind
   brew update-python-resources llmind-cli
   ```

   This rewrites the `BEGIN AUTOGENERATED RESOURCES` block with pinned
   `resource` declarations (URL + SHA256) for every transitive Python
   dependency. Commit the result.

5. **Audit and test the formula** before pushing the tap update:

   ```bash
   brew audit --strict --online dmitryrollins/llmind/llmind-cli
   brew install --build-from-source dmitryrollins/llmind/llmind-cli
   brew test dmitryrollins/llmind/llmind-cli
   ```

## What's bundled

The default formula installs the CLI plus the three cloud providers
(`anthropic`, `openai`, `google-genai`) so `brew install` produces a working
binary.

`whisper-local` is **not** bundled â€” `faster-whisper` pulls in CTranslate2
and large model downloads. Users who want local transcription can install it
into the Homebrew-managed virtualenv after the fact:

```bash
"$(brew --prefix llmind-cli)/libexec/bin/pip" install faster-whisper
```

If demand for a bundled local-whisper variant grows, add a sibling formula
`llmind-cli-whisper` with that resource included. Don't use formula options
â€” Homebrew has deprecated them.

## CI automation

[`.github/workflows/release-homebrew.yml`](../../.github/workflows/release-homebrew.yml)
runs on every `cli-v*` tag push (and via manual `workflow_dispatch`). It:

1. Computes the source tarball SHA256.
2. Checks out the tap repo into the Homebrew prefix.
3. Updates `url`, `sha256`, `version` in `Formula/llmind-cli.rb`.
4. Runs `brew update-python-resources` to regenerate the resource block.
5. Runs `brew audit --strict --online` and a `brew install --build-from-source`
   + `brew test` smoke test.
6. Opens a PR in the tap repo with the changes.

### One-time setup before the workflow can run

1. Create a **fine-grained Personal Access Token** scoped to
   `dmitryrollins/homebrew-llmind` with these permissions:
   - **Contents**: Read and write
   - **Pull requests**: Read and write
2. In this repo (`dmitryrollins/LLMind`), go to **Settings â†’ Secrets and
   variables â†’ Actions** and add it as `HOMEBREW_TAP_TOKEN`.
3. Make sure the tap repo has a `main` branch (default for new repos).

After that, every `git push origin cli-v0.1.0` produces a PR in the tap
within a few minutes. Review and merge to publish.

### First-release checklist

For the very first release, run the manual steps once to confirm the formula
works end-to-end:

1. Tag and release `cli-v0.1.0`.
2. Trigger the workflow via `workflow_dispatch` with input `cli-v0.1.0`
   (or just push the tag).
3. Verify the PR opened in the tap repo, merge it.
4. From a clean shell on macOS:
   ```bash
   brew tap dmitryrollins/llmind
   brew install llmind-cli
   llmind --version
   ```
